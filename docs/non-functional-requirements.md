# Non-Functional Requirements – DriveAlb

## Performance
NFR-01: The system should load main pages within an acceptable time under normal usage conditions.  
NFR-02: Search and filter operations should return results quickly.  
NFR-03: The system should support multiple users accessing the platform at the same time.  

## Usability
NFR-04: The user interface should be simple, clear, and easy to navigate.  
NFR-05: The platform should be usable by users with basic computer knowledge.  
NFR-06: The design should be responsive and accessible on desktop and mobile devices.  

## Security
NFR-07: User passwords shall be stored securely in encrypted or hashed form.  
NFR-08: The system shall protect authenticated pages from unauthorized access.  
NFR-09: The system should validate user input to reduce security risks.  
NFR-10: The system should protect user and listing data from unauthorized modification.  

## Reliability
NFR-11: The system should operate consistently without frequent crashes.  
NFR-12: The system should handle invalid input gracefully and display meaningful error messages.  
NFR-13: The system should preserve data integrity during create, update, and delete operations.  

## Maintainability
NFR-14: The codebase should be organized clearly to support future updates.  
NFR-15: The system should be modular so that frontend and backend components can be maintained separately.  
NFR-16: Documentation should be available for important project features and setup steps.  

## Scalability
NFR-17: The system should be designed so that new features can be added in the future.  
NFR-18: The database structure should support growth in the number of users and listings.  

## Compatibility
NFR-19: The platform should work correctly on modern web browsers.  
NFR-20: The system should support deployment on common development and hosting environments.
