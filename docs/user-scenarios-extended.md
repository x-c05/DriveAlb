# User Scenarios Extended – DriveAlb

## 1. Buyer browses available cars
A buyer enters the DriveAlb platform and views all available car listings shown on the homepage or listings page. The buyer can scroll through different cars and get a general overview of the options available.

## 2. Buyer searches for a specific car
A buyer enters a keyword such as a brand, model, or car type in the search bar. The system displays listings that match the entered term.

## 3. Buyer filters car listings
A buyer applies filters such as price range, brand, model, year, transmission, or fuel type. The platform updates the displayed cars according to the selected filters.

## 4. Buyer views car details
A buyer clicks on a car listing to open the detailed page. The page shows information such as model, brand, year, mileage, price, description, and images.

## 5. Buyer contacts seller
A buyer decides that they are interested in a vehicle and uses the available contact option to reach the seller for more information or to arrange a meeting.

## 6. Seller creates an account
A seller opens the registration page, enters personal information, and creates an account. After successful registration, the seller can access seller functions.

## 7. Seller logs in
A registered seller enters their credentials and logs in. The system verifies the credentials and redirects the seller to their dashboard.

## 8. Seller posts a car listing
A seller fills in the required listing form, including car details such as brand, model, year, price, fuel type, and description. After submitting the form, the new listing becomes visible on the platform.

## 9. Seller uploads car images
While creating or editing a listing, the seller uploads one or more images of the car. The system stores the images and connects them to the selected listing.

## 10. Seller edits a listing
A seller opens one of their existing listings and updates information such as price, description, or car status. The changes are saved and reflected on the platform.

## 11. Seller deletes a listing
A seller removes a listing that is no longer available or relevant. The system deletes or hides the listing from public view.

## 12. Admin manages users
An admin accesses the admin panel and views user accounts. The admin can inspect user details and take actions when necessary.

## 13. Admin manages listings
An admin reviews all listings on the platform. If a listing is fraudulent, inappropriate, or violates platform rules, the admin can remove it.

## 14. System enforces role-based access
The system ensures that only sellers can create listings, only admins can access admin features, and buyers can browse listings without unauthorized access to restricted functions.

## 15. System stores and retrieves data
The platform stores user data, listings, and images in a database. When a user performs an action, the system retrieves or updates the relevant data.
