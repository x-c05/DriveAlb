# User Scenarios List – DriveAlb

## Buyer Scenarios
1. A buyer visits the platform and browses available car listings.
2. A buyer searches for a specific car by keyword.
3. A buyer filters cars by brand, model, year, price, and fuel type.
4. A buyer views detailed information about a selected car.
5. A buyer contacts the seller for more information.
6. A buyer creates an account to save preferences or interact with listings.

## Seller Scenarios
7. A seller registers for an account.
8. A seller logs into the platform.
9. A seller creates a new car listing.
10. A seller uploads images for a car listing.
11. A seller edits an existing car listing.
12. A seller deletes a listing that is no longer needed.
13. A seller views all of their posted listings.

## Admin Scenarios
14. An admin logs into the administration panel.
15. An admin views all users on the platform.
16. An admin reviews and manages car listings.
17. An admin removes inappropriate or fraudulent listings.
18. An admin suspends or manages user accounts.
19. An admin monitors platform activity.

## System Scenarios
20. The system validates user login credentials.
21. The system stores car listings in the database.
22. The system displays search results based on user filters.
23. The system restricts access based on user roles.
24. The system handles image uploads securely.
