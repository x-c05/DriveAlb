# Case Study – DriveAlb

## 1. Introduction

DriveAlb is a web-based platform designed to simplify the process of buying and selling cars in Albania. The platform aims to provide a centralized, secure, and user-friendly environment where users can browse, list, and manage car advertisements.

The system addresses the need for a more organized and efficient marketplace compared to existing informal channels.

---

## 2. Problem Statement

Currently, many people in Albania buy and sell cars through social media platforms or unstructured websites. This creates several problems:

* Information is often incomplete or unreliable
* It is difficult to compare different vehicles
* Search and filtering options are limited
* There is no standardized system for managing listings

These challenges lead to a poor user experience and increase the risk of misinformation.

---

## 3. Objectives

The main objectives of DriveAlb are:

* To create a centralized platform for car listings
* To improve transparency and accessibility of information
* To allow users to easily search and filter vehicles
* To provide a secure system for managing user accounts and listings

---

## 4. Proposed Solution

DriveAlb will offer a web application where:

* Buyers can browse and filter car listings
* Sellers can create and manage their own listings
* Users can view detailed information for each vehicle

The platform will be designed to be intuitive, responsive, and accessible from different devices.

---

## 5. Stakeholders

The main stakeholders of the system include:

* Buyers (individual users searching for cars)
* Sellers (individuals or businesses offering cars)
* System administrators (responsible for maintaining the platform)

---

## 6. System Features

### Buyer Features

* Browse all available cars
* Search and filter by criteria (price, brand, year, etc.)
* View detailed car descriptions

### Seller Features

* Register and log in
* Create, edit, and delete car listings
* Upload images and provide specifications

### Admin Features

* Manage user accounts
* Monitor listings
* Remove inappropriate or fraudulent content

---

## 7. Functional Requirements

* User registration and authentication
* Creation and management of car listings
* Search and filtering functionality
* Role-based access control (buyer, seller, admin)

---

## 8. Non-Functional Requirements

* Usability: simple and intuitive interface
* Performance: fast loading and response times
* Security: secure authentication and data protection
* Scalability: ability to handle increased users and listings

---

## 9. Technology Stack (Planned)

* Frontend: React.js
* Backend: Node.js
* Database: MySQL
* Version Control: Git & GitHub

---

## 10. Conclusion

DriveAlb aims to modernize the car marketplace in Albania by providing a reliable, structured, and efficient platform. By addressing the limitations of existing solutions, the system will enhance user experience and improve trust in online car transactions.
